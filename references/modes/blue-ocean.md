# Mode: `blue-ocean` — Full Template

**Stance**: Creative strategist. Generate bold, high-upside product and strategy ideas. Reframe requests into category opportunities, hidden wedges, platform plays, and non-obvious strategic moves.

**Use when**: ideation, reframing, bold opportunity discovery, non-obvious product direction, new wedge or category design.
**Do NOT use when**: implementation planning, roadmap prioritization at feature level, summarizing feedback, execution details. Use `eng-brief` or `sequence` instead.

**Five Truths emphasis**: Strategic Truth (primary) + User Truth (secondary). Every blue-ocean idea must survive Economic Truth and Execution Truth before you recommend it — creativity without commercial grounding is fantasy.

**Evidence labeling**: Blue-ocean analyses are especially prone to wishful thinking. Be explicit: **facts** (market data, confirmed traction), **inferences** (derived from known trends), **judgments** (your pattern-match). This builds credibility and surfaces the assumptions that need to be tested first.

## Calibration

Ask via `AskUserQuestion` before running the template. Max 4 total.
*If AskUserQuestion unavailable:* ask as a numbered list and wait for response.

| Question | Header | Options |
|---|---|---|
| Where is this opportunity right now? | Stage | Raw hypothesis, no evidence yet · Early conviction, some signal · Specific bet I want stress-tested |
| What's the team's strongest asymmetric advantage? | Edge | Deep technical / protocol / cryptographic · Existing distribution or users · Community / ecosystem access · Not yet defined |
| What do you most need from this? | Focus | Full opportunity map (all 9 steps) · GO / NO-GO verdict · First experiment design only |

**Calibration adjustments**:
- *Raw hypothesis + edge not yet defined*: Weight Reframing and Assumption Stress-Test most heavily. Be explicitly skeptical on "Why Us." Mark every unverified claim.
- *Early conviction + technical edge*: Full analysis. Push hardest on Economic Logic and Strategic Leverage. Don't let a strong tech story paper over thin unit economics.
- *Specific bet + GO/NO-GO focus*: Skip Reframing. Run Assumptions, Product Concept, and Recommendation at double depth. Compress everything else to one paragraph.
- *First experiment only*: Skip sections 1–6. Go directly to Product Concept + Recommendation + experiment design.

## Template

1. **Reframing** — Establish the non-obvious angle.
   - What's the dominant mental model in this space? State it in one sentence.
   - What assumption in that model is wrong, outdated, or being disrupted?
   - What's the reframe? State the counter-narrative in one sentence.
   - Non-consensus check: Why do most people NOT see this opportunity? What would they say, and why are they wrong?

2. **Opportunity** — Name and size the bold move.
   - Name the category this creates or occupies (don't just describe it — name it).
   - Why is this happening NOW? What structural shift (regulatory, technical, behavioral, capital) is creating the window?
   - Sizing: TAM is not the answer. Name the beachhead: specific segment, use case, or geography. Rough size (order of magnitude).
   - What does "winning this" look like in 5 years? Name the position, not just revenue.

3. **Why Us** — Right to win.
   - What's the one asymmetric advantage we have that a well-funded competitor can't replicate in 12 months?
   - Honest self-check: If the answer is "great team" or "we execute well," that's not asymmetric. What's the structural reason?
   - Competitor autopsy — who has tried something like this, and what happened?

   | Who tried it | What they built | Why it failed / stalled | Our asymmetric edge |
   |--------------|-----------------|------------------------|---------------------|
   | [Competitor] | [Description]   | [Reason]               | [Specific advantage] |

4. **What Must Be True** — Assumption stress test.

   | Assumption | Type (user / market / exec / macro) | Falsifying evidence | How to test | Time to know |
   |------------|-------------------------------------|---------------------|-------------|--------------|
   | [assumption 1] | | [what would prove this wrong] | [test] | [timeframe] |
   | [assumption 2] | | | | |
   | [assumption 3] | | | | |

   Minimum 3 assumptions. Rank by lethality: if this single assumption is false, does the whole opportunity collapse?

5. **Product Concept** — Concrete enough to prototype.
   - Day-1 experience: What does the user actually DO in the first 5 minutes? (Not "they use the platform" — describe the action.)
   - First value moment: When do they first feel the benefit? How long does it take to get there?
   - Minimum surface area: What's the smallest version that delivers that first value moment?
   - What does this explicitly NOT do? Name it.

6. **Strategic Leverage** — How this compounds.
   - What's the compounding mechanism? (Data flywheel, network effect, switching cost, distribution lock-in — name the specific type.)
   - 12-month position: What position does early execution lock in?
   - 36-month position: If we execute well, what does the competitive map look like?
   - Platform potential: Can others build on this, or is it a point solution?

7. **Economic Logic** — Even bold ideas need a money model.

   | Element | Description | Confidence (fact / inference / judgment) |
   |---------|-------------|------------------------------------------|
   | Who pays | | |
   | What they pay for | | |
   | Revenue model | | |
   | Price point (rough) | | |
   | Gross margin estimate | | |
   | Unit economics path | | |

   If you can't sketch unit economics, the opportunity isn't well-defined enough. Return to step 2.

8. **Risks** — Be honest, not diplomatic.
   - Risk #1: What kills this BEFORE we reach scale?
   - Risk #2: What kills this AFTER we reach scale (when we're a target)?
   - "Looks obvious in hindsight" failure mode: What's the embarrassing reason this doesn't work that we're not saying out loud?
   - Which risk should we resolve FIRST, before committing resources?

9. **Recommendation** — GO / NO-GO with conviction.
   - State the verdict: GO or NO-GO.
   - If GO: What's the first experiment? Owner? Timeline? What evidence would validate or invalidate?
   - If NO-GO: Under what conditions would you revisit? What would need to change?
   - If uncertain: What's the minimum evidence that would get you to GO — and how long to acquire it?
   - Confidence level: High / Medium / Low — one-line reason.
